正如我们之前讨论的，它把列表中的每一个元素（无论这个元素是数字、向量还是数据框）拿出来检验，保留满足条件的子元素。
```R
my_list <- list(a = 1:3, b = "hello", c = 4:6)
# 保留长度大于 2 的元素
keep(my_list, ~ length(.x) > 2)
# 返回: list(a = 1:3, c = 4:6)
```

#### 3. 作用于“数据框” (Data Frame) -> ⚠️ 按**列**过滤（极易踩坑！）
**高能预警**：因为数据框在 R 底层本质上是一个**列表（List of vectors）**，所以 `keep()` 作用于数据框时，**它是按“列”来过滤的，而不是按“行”！** 传入 `.x` 的是每一列的数据向量。
```R
# 保留均值大于 20 的“列”
keep(mtcars, ~ mean(.x) > 20) 
# 返回的是一个只包含 mpg, cyl, disp 等均值大于20的列的 data.frame。
# 它绝对不会帮你过滤掉 mpg < 20 的“行”！
```
*(注：如果你想对数据框按**行**过滤，请看后文的 `dplyr::filter`。)*

### 二、 `keep()` 的相关函数家族

在 `purrr` 和整个 `tidyverse` 生态中，围绕“条件筛选”有一整套设计极其优雅的函数矩阵。

#### 1. `purrr` 家族内部的“兄弟姐妹”

| 函数 | 作用 | 与 `keep()` 的关系 | 代码示例 |
| :--- | :--- | :--- | :--- |
| **`discard()`** | **丢弃**满足条件的元素 | `keep()` 的**反义词**。 | `discard(1:10, ~ .x > 5)` <br>返回 `1:5` |
| **`compact()`** | 丢弃 `NULL` 或空值 | `keep()` 的**特化版**。等价于 `keep(x, ~ length(.x) > 0)` 或 `discard(x, is.null)`。 | `compact(list(1, NULL, 2, list()))` <br>返回 `list(1, 2)` |
| **`detect()`** | 找到**第一个**满足条件的元素 | `keep()` 的**短路版**。只返回第一个匹配的元素本身，找不到返回 `NULL`。 | `detect(1:10, ~ .x > 5)` <br>返回 `6` |
| **`detect_index()`**| 找到**第一个**满足条件的元素的**索引** | 返回位置索引，找不到返回 `0`。 | `detect_index(1:10, ~ .x > 5)` <br>返回 `6` |
| **`every()`** | 判断是否**所有**元素都满足条件 | 返回单个逻辑值 (`TRUE`/`FALSE`)。 | `every(1:10, ~ .x > 0)` <br>返回 `TRUE` |
| **`some()`** | 判断是否**至少有一个**元素满足条件 | 返回单个逻辑值 (`TRUE`/`FALSE`)。 | `some(1:10, ~ .x > 5)` <br>返回 `TRUE` |

#### 2. 基础 R (Base R) 中的“平替”

如果你不想加载 `purrr`，基础 R 也有完全等价的实现：

*   **`Filter()`**：基础 R 中完美等价于 `purrr::keep()` 的函数。
    ```R
    # 基础 R 写法
    Filter(function(x) x > 5, 1:10) 
    # 等价于 purrr::keep(1:10, ~ .x > 5)
    ```
*   **逻辑索引 (Logical Indexing)**：这是 R 语言最底层的过滤逻辑，`keep()` 底层其实就是这么干的。
    ```R
    nums <- 1:10
    nums[nums > 5] 
    ```

#### 3. `dplyr` 家族：数据框操作的“正规军”

如前所述，`keep()` 处理数据框时是**按列**过滤的。如果你需要对数据框进行常规的行列过滤，`dplyr` 提供了更语义化的函数：

*   **按行过滤（对应向量的 keep）**：使用 **`dplyr::filter()`**
    ```R
    library(dplyr)
    # 保留 mpg > 20 的“行”
    mtcars %>% filter(mpg > 20) 
    ```
*   **按列过滤（对应列表的 keep）**：使用 **`dplyr::select()` 结合 `where()`**
    ```R
    # 保留所有数值型的“列” (极其强大的类型过滤)
    mtcars %>% select(where(is.numeric))
    
    # 保留均值大于 20 的“列” (等价于 keep(mtcars, ~ mean(.x) > 20))
    mtcars %>% select(where(~ mean(.x) > 20))
    ```

### 三、 总结与选型指南

在实际写代码时，面对“筛选”需求，应该选哪个函数？请参考以下决策树：

1.  **你要处理的是“列表 (List)”吗？**
    *   是 $\rightarrow$ 毫不犹豫使用 **`purrr::keep()`** 或 **`purrr::discard()`**。
    *   只是想清理掉列表里的 `NULL` $\rightarrow$ 使用 **`purrr::compact()`**。

2.  **你要处理的是“数据框 (Data Frame / Tibble)”吗？**
    *   想过滤**行** $\rightarrow$ 使用 **`dplyr::filter()`**（千万别用 `keep`，会按列过滤）。
    *   想根据**列的属性/类型**过滤列 $\rightarrow$ 使用 **`dplyr::select(where(...))`**。

3.  **你要处理的是“原子向量 (Vector)”吗？**
    *   是 $\rightarrow$ 使用 **`purrr::keep()`**，或者更原生的 **逻辑索引 `x[x > 5]`**（后者在简单场景下速度更快、更常见）。

4.  **你只需要找“第一个”符合条件的，或者判断“有没有”符合条件的？**
    *   找第一个元素 $\rightarrow$ **`purrr::detect()`**
    *   找第一个索引 $\rightarrow$ **`purrr::detect_index()`**
    *   判断真假 $\rightarrow$ **`purrr::every()` / `purrr::some()`**

掌握这个函数矩阵，你在 R 语言中处理任何“条件筛选”的需求时，都能写出最地道、最高效的代码。
